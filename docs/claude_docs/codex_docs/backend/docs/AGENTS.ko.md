# Codex 프로젝트 메모

## 기본 응답 방식
- 기본적으로 한국어로 응답한다.

## 문서 우선 확인
- 새로운 세션이 시작되면 저장소 전반의 백엔드 또는 제품 요청을 처리하기 전에 `docs/` 아래의 마크다운 문서를 먼저 확인한다.
- `docs/`의 문서들은 선택 참고자료가 아니라 기본 프로젝트 컨텍스트로 취급한다.
- 요청 범위가 분명하면 관련 문서를 우선 읽고, 요청이 넓거나 애매하면 구현 전에 `docs/` 트리를 전체적으로 훑는다.
- 기존 추정과 `docs/` 문서 내용이 충돌하면 문서 기준을 따른다.

## 커밋 규칙
- 사용자가 구현 작업을 요청했고 별도로 커밋하지 말라고 하지 않았다면 작업 완료 후 자동으로 커밋한다.
- 커밋 메시지는 한글로 작성한다.
- 사용자가 한 번에 서로 다른 기능이나 변경사항을 여러 개 요청했다면, 가능하면 기능 단위로 나눠서 각각 별도 커밋한다.

## 프론트엔드 관련 문서
- 프론트엔드 전용 작업 규칙은 `web/docs/AGENTS.md`에 둔다.
- 프론트엔드 전용 한글 문서는 `web/docs/AGENTS.ko.md`에 둔다.
